# DIP 影像處理演算法技術文件 (docs.md)

本文件詳細彙整專案中已實作完成之影像處理核心演算法原理、參數說明與具體實作方法。所有公式皆以標準 ASCII 純文字表示。

---

## 1. 影像轉灰階 (encode_gray)
* **演算法原理**：
  人眼對不同顏色的感受亮度不同。BT.601 標準公式依據人眼生理特性，給予紅(R)、綠(G)、藍(B)三個通道不同的敏感度加權係數。計算公式為：
  `Y = 0.299 * R + 0.587 * G + 0.114 * B`
  其中 Y 代表亮度（灰階值）。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int): 影像寬度（像素）。
  - `h` (int): 影像高度（像素）。
  - `d` (int): 影像通道數（d=3 代表 BGR 彩色，d=1 代表灰階直接拷貝）。
  - `g` (int*): 輸出影像陣列指標。
* **實作方法**：
  若影像通道為 `d=3`，遍歷所有像素並按索引提取 B、G、R 通道值，計算加權亮度值 `gray_val`，並將其填入輸出影像對應像素的 B、G、R 通道中，使影像呈現灰色；若 `d=1`，則執行直接複製。

## 2. 位元平面切片 (bit_plane_slice)
* **演算法原理**：
  影像中每個 8-bit 灰階像素是由 8 個二進位位元組成。第 7 位元（最高位元，MSB）所佔權重最大（128），代表最顯著的影像結構與輪廓；第 0 位元（最低位元，LSB）所佔權重最小（1），代表精細紋理或噪訊。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `plane` (int): 欲提取的目標位元平面編號（範圍為 0 到 7）。
  - `binarize` (int): 是否將輸出值二值化放大（1: 放大至 255，0: 保留原始權重）。
* **實作方法**：
  先計算遮罩 `mask = 1 << plane`。遍歷影像的每個像素，進行位元與運算 `bitVal = pixel & mask`。當 `binarize != 0` 時，非零值映射為 255，零值映射為 0；當 `binarize == 0` 時，直接輸出 `bitVal`。支援 1 通道與 3 通道 BGR 影像。

## 3. 統計直方圖數據 (calculate_histogram)
* **演算法原理**：
  直方圖代表影像中各個亮度值（0-255）在各通道中出現的頻率。它是許多影像增強與分割演算法（如直方圖等化、大津法）的基礎統計數據。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數（d=1 代表單通道，d=3 代表 BGR，d=4 代表 BGRA）。
  - `histB` (int*): 存放藍色通道（或單通道灰階）的直方圖陣列指標（大小為 256）。
  - `histG` (int*): 存放綠色通道直方圖指標（大小為 256）。
  - `histR` (int*): 存放紅色通道直方圖指標（大小為 256）。
* **實作方法**：
  先將輸入的直方圖陣列清零。當 `d=1` 時，遍歷每個像素取得灰階亮度，累加至 `histB` 陣列。當 `d=3` 或 `d=4` 時，提取 B、G、R 通道值並各自累加；若為 `d=4` 且像素 Alpha 通道為 0（全透明），則跳過該像素以避免統計透明背景干擾。

## 4. 直方圖等化 (histogram_equalization)
* **演算法原理**：
  藉由展寬影像中集中的灰階分佈以達到增強局部對比度的效果。其數學基礎是將原直方圖亮度分佈轉換為均勻分佈。計算公式為累積機率分佈函數 (CDF, Cumulative Distribution Function)：
  `LUT[i] = round( (CDF[i] / TotalPixels) * 255 )`
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數（1 代表灰階，3 代表 BGR 彩色）。
  - `g` (int*): 輸出影像陣列指標。
* **實作方法**：
  1. 統計影像各通道亮度直方圖。
  2. 計算累積直方圖，即亮度 `0` 至 `i` 的像素個數總和 `sum`。
  3. 根據累積總和計算查找表 `lut[i] = (int)(sum * 255.0 / (w * h) + 0.5)`，並限制在 `[0, 255]` 區間。
  4. 遍歷影像，將所有像素的灰階值經由對應通道的 `lut` 進行重新映射。

## 5. 亮度對比與 Gamma 調整 (adjust_brightness_contrast)
* **演算法原理**：
  - **線性調整**：利用線性算式改變亮度與對比度：
    `Y = alpha * X + beta`
    當對比係數 `alpha >= 0` 時，`alpha` 控制對比度斜率，`beta` 控制整體亮度平移。
  - **非線性 Gamma 調整**：修正相機感光或顯示器非線性特性的冪律公式：
    `Y = 255 * (X / 255) ^ gamma`
    其中 `gamma = -alpha`。當 `gamma < 1` 暗部拉伸，影像變亮；`gamma > 1` 亮部壓縮，影像變暗。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `alpha` (double): 大於等於 0 時為對比度係數；小於 0 時 `-alpha` 代表非線性 Gamma 校正係數。
  - `beta` (int): 線性亮度增量參數（範圍 -255 到 +255）。
* **實作方法**：
  為避免在數百萬像素的影像上重複執行昂貴的冪次或乘加運算，演算法在遍歷像素前，先將 0-255 共 256 種亮度值的變換結果預先計算好並存入查找表 `lut` 中。最後只需對影像像素值進行 `O(1)` 的 `lut` 查表操作。

## 6. 影像旋轉 (rotate_image)
* **演算法原理**：
  影像旋轉使用旋轉變換矩陣來映射新舊座標。為避免正向投影（原圖投影至新圖）因座標四捨五入而產生畫布空洞，本演算法採用反向映射（從目標圖像反向計算原圖對應坐標）：
  `src_x = dx * cos(theta) - dy * sin(theta) + cx_src`
  `src_y = dx * sin(theta) + dy * cos(theta) + cy_src`
  其中 `dx = xp - cx_dst`，`dy = yp - cy_dst`，`theta` 為旋轉弧度。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 原始影像寬度與高度。
  - `d` (int): 影像通道數（1, 3 或 4）。
  - `g` (int*): 輸出影像陣列指標。
  - `newW` (int), `newH` (int): 旋轉後的新畫布寬度與高度。
  - `angle_deg` (double): 旋轉角度（單位度，順時針為正值）。
  - `mode` (int): 插值映射模式（0: 最近鄰反向, 1: 雙線性反向, 2: 正向映射）。
  - `bg_r`, `bg_g`, `bg_b`, `bg_a` (int): 出界區域所要填充的背景顏色 (RGBA)。
* **實作方法**：
  根據角度預先計算 `cos(theta)` 與 `sin(theta)`。遍歷新畫布的每個像素 `(xp, yp)`，計算出原圖實數坐標 `(src_x, src_y)`。
  - **最近鄰插值**：對坐標直接四捨五入，取得對應原圖像素。
  - **雙線性插值**：取得四周相鄰的 4 個像素坐標，依據水平與垂直小數距離進行距離權重乘加計算：
    `w00 = (1-a)*(1-b), w10 = a*(1-b), w01 = (1-a)*b, w11 = a*b`
    進行內插值混色，消除旋轉後物體邊緣的鋸齒。超出邊界的像素則以給定背景色填充。

## 7. 幾何縮放 (scale_image)
* **演算法原理**：
  改變影像寬度與高度。縮放時連續坐標映射為：`src_coord = ((dst_coord + 0.5) * scale) - 0.5`。
  - **最近鄰插值**：將對應的實數坐標向下取整加 0.5（四捨五入），取得最靠近的像素值。
  - **雙線性插值**：藉由目標座標對應原圖的實數坐標，利用其四周 4 個相鄰像素值進行雙線性加權計算。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `newW` (int), `newH` (int): 縮放後的新寬度與新高度。
  - `mode` (int): 縮放插值模式（0: 最近鄰, 1: 雙線性）。
* **實作方法**：
  藉由 `scaleX = w / newW` 與 `scaleY = h / newH` 計算比例。實作中採用「中心補償對齊」，避免幾何中心對齊漂移。對原圖坐標採取 Strict Boundary Clamp：`iy` 限制在 `[0, h - 1]` 區間，`ix` 限制在 `[0, w - 1]` 區間，防範像素訪問越界引起崩潰。

## 8. 手動門檻二值化 (manual_threshold)
* **演算法原理**：
  基於單一閥值 T 的二值分割方法。將多階灰階值轉為純黑白（0 或 255）兩級。
* **參數說明**
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `T` (int): 使用者指定的分隔門檻值（0 到 255）。
* **實作方法**：
  遍歷每個像素。若影像為 `d=1`，當 `pixel_val >= T` 輸出 255，否則輸出 0。若影像為 `d=3` 或 `d=4`，先以 BT.601 公式計算其灰階亮度值，再與 `T` 進行比較，並將 B、G、R 三個通道的值同時設為 0 或 255；若是 `d=4`，則保持原有的 Alpha 透明度值以維護透明背景。

## 9. 大津法二值化 (otsu_threshold)
* **演算法原理**：
  一種自適應二值化方法。其數學準則是透過分析直方圖，自動尋找一個最佳閾值 T_best，使得分割後的背景類別 C1（灰階小於閾值）與前景類別 C2（灰階大於等於閾值）之間的**類間變異數**最大。
  累積背景像素機率權重 `w1(t)`，前景為 `w2(t) = 1 - w1(t)`。
  背景灰階均值 `mu1(t)`，前景灰階均值 `mu2(t)`。
  類間變異數公式：
  `sigma_b^2(t) = w1(t) * w2(t) * (mu1(t) - mu2(t))^2`
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
* **實作方法**：
  1. 統計影像直方圖，並計算所有像素灰階值的加權總和 `total_sum`。
  2. 迴圈疊代 t 從 0 到 254，動態維護背景的累積像素個數 `w_bg` 與背景累積灰階和 `sum_bg`。
  3. 當前閾值 t 下的背景均值為 `mean_bg = sum_bg / w_bg`，前景均值為 `mean_fg = (total_sum - sum_bg) / (TotalPixels - w_bg)`。
  4. 計算當前的類間變異數。若大於歷史最大變異數，則更新最大變異數值，並將 `best_threshold` 設為 t。
  5. 疊代結束後，使用最佳閾值 `best_threshold` 對影像像素執行二值化（0 或 255）映射，輸出至 `g` 中（4 通道保留原有 Alpha）。
