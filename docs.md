# DIP 影像處理演算法技術文件 (docs.md)

本文件詳細彙整專案中已實作完成之影像處理核心演算法原理、參數說明與具體實作方法。所有公式皆以標準 ASCII 純文字表示。

---

## 1. 影像轉灰階 (encode_gray / ConvertTo8bppGrayscale)
* **演算法原理**：
  人眼對不同顏色的感受亮度不同。標準公式依據人眼生理特性，給予紅(R)、綠(G)、藍(B)三個通道不同的敏感度加權係數（ITU-R BT.601 標準）：
  `Y = 0.299 * R + 0.587 * G + 0.114 * B`
  其中 Y 代表亮度（灰階值）。本專案提供兩個子功能：
  1. **灰階顯示預覽 (24-bit)**：在 BGR 三通道中填入相同的 Y 值，輸出仍為 24-bit 格式，便於一般顯示與預覽。
  2. **轉換為灰階圖像 (8-bit)**：建立 Format8bppIndexed 格式的點陣圖，並設定線性漸變的 256 色灰階調色盤，將像素的 Y 值寫入 8-bit 索引中。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int): 影像寬度（像素）。
  - `h` (int): 影像高度（像素）。
  - `d` (int): 影像通道數（d=3 代表 BGR 彩色，d=1 代表灰階直接拷貝）。
  - `g` (int*): 輸出影像陣列指標。
* **實作方法**：
  - 24-bit 灰階轉換調用 C++ `encode_gray` 函式，直接在 $d=3$ 的原格式通道中填入計算後的 Y 值。
  - 8-bit 灰階轉換在 C# 層呼叫 `ConvertTo8bppGrayscale`，建立標準 Format8bppIndexed 影像並鎖定記憶體，將各像素的 Y 值填入其 8-bit 索引位元組中，並為其綁定線性灰階調色盤。
  - **直方圖數值差異說明**：由於 24-bit 灰階直接儲存真實的 BGR 通道數值，而 8-bit 灰階則將數值儲存為指向調色盤的索引值。在 GDI+ 色彩管理、浮點數捨入誤差（如 C++ 截斷與 C# 的逼近）以及調色盤對齊的共同影響下，兩者在部分影像的直方圖分佈上可能產生極微小的數值偏差，此屬正常現象。

## 2. 位元平面切片 (bit_plane_slice)
* **演算法原理**：
  影像中每個 8-bit 灰階像素是由 8 個二進位位元組成。第 7 位元（最高位元，MSB）所佔權重最大（128），代表最顯著的影像結構與輪廓；第 0 位元（最低位元，LSB）所佔權重最小（1），代表精細紋理或噪訊。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `plane` (int): 欲提取的目標位元平面編號（範圍為 0 到 7）。
  - `binarize` (int): 是否將輸出值二值化放大（1: 放大至 255，0: 保留原始權重）。
* **實作方法**：
  先計算遮罩 `mask = 1 << plane`。遍歷影像的每個像素，進行位元與運算 `bitVal = pixel & mask`。當 `binarize != 0` 時，非零值映射為 255，零值映射為 0；當 `binarize == 0` 時，直接輸出 `bitVal`。支援 1 通道與 3 通道 BGR 影像。

## 3. 統計直方圖數據 (calculate_histogram)
* **演算法原理**：
  直方圖代表影像中各個亮度值（0-255）在各通道中出現的頻率。它是許多影像增強與分割演算法（如直方圖等化、大津法）的基礎統計數據。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數（d=1 代表單通道，d=3 代表 BGR，d=4 代表 BGRA）。
  - `histB` (int*): 存放藍色通道（或單通道灰階）的直方圖陣列指標（大小為 256）。
  - `histG` (int*): 存放綠色通道直方圖指標（大小為 256）。
  - `histR` (int*): 存放紅色通道直方圖指標（大小為 256）。
* **實作方法**：
  先將輸入的直方圖陣列清零。當 `d=1` 時，遍歷每個像素取得灰階亮度，累加至 `histB` 陣列。當 `d=3` 或 `d=4` 時，提取 B、G、R 通道值並各自累加；若為 `d=4` 且像素 Alpha 通道為 0（全透明），則跳過該像素以避免統計透明背景干擾。

## 4. 直方圖等化 (histogram_equalization)
* **演算法原理**：
  藉由展寬影像中集中的灰階分佈以達到增強局部對比度的效果。其數學基礎是將原直方圖亮度分佈轉換為均勻分佈。計算公式為累積機率分佈函數 (CDF, Cumulative Distribution Function)：
  `LUT[i] = round( (CDF[i] / TotalPixels) * 255 )`
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數（1 代表灰階，3 代表 BGR 彩色）。
  - `g` (int*): 輸出影像陣列指標。
* **實作方法**：
  1. 統計影像各通道亮度直方圖。
  2. 計算累積直方圖，即亮度 `0` 至 `i` 的像素個數總和 `sum`。
  3. 根據累積總和計算查找表 `lut[i] = (int)(sum * 255.0 / (w * h) + 0.5)`，並限制在 `[0, 255]` 區間。
  4. 遍歷影像，將所有像素的灰階值經由對應通道的 `lut` 進行重新映射。

## 5. 亮度對比與 Gamma 調整 (adjust_brightness_contrast)
* **演算法原理**：
  - **線性調整**：利用線性算式改變亮度與對比度：
    `Y = alpha * X + beta`
    當對比係數 `alpha >= 0` 時，`alpha` 控制對比度斜率，`beta` 控制整體亮度平移。
  - **非線性 Gamma 調整**：修正相機感光或顯示器非線性特性的冪律公式：
    `Y = 255 * (X / 255) ^ gamma`
    其中 `gamma = -alpha`。當 `gamma < 1` 暗部拉伸，影像變亮；`gamma > 1` 亮部壓縮，影像變暗。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `alpha` (double): 大於等於 0 時為對比度係數；小於 0 時 `-alpha` 代表非線性 Gamma 校正係數。
  - `beta` (int): 線性亮度增量參數（範圍 -255 到 +255）。
* **實作方法**：
  為避免在數百萬像素的影像上重複執行昂貴的冪次或乘加運算，演算法在遍歷像素前，先將 0-255 共 256 種亮度值的變換結果預先計算好並存入查找表 `lut` 中。最後只需對影像像素值進行 `O(1)` 的 `lut` 查表操作。

## 6. 影像旋轉 (rotate_image)
* **演算法原理**：
  影像旋轉使用旋轉變換矩陣來映射新舊座標。為避免正向投影（原圖投影至新圖）因座標四捨五入而產生畫布空洞，本演算法採用反向映射（從目標圖像反向計算原圖對應坐標）：
  `src_x = dx * cos(theta) - dy * sin(theta) + cx_src`
  `src_y = dx * sin(theta) + dy * cos(theta) + cy_src`
  其中 `dx = xp - cx_dst`，`dy = yp - cy_dst`，`theta` 為旋轉弧度。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 原始影像寬度與高度。
  - `d` (int): 影像通道數（1, 3 或 4）。
  - `g` (int*): 輸出影像陣列指標。
  - `newW` (int), `newH` (int): 旋轉後的新畫布寬度與高度。
  - `angle_deg` (double): 旋轉角度（單位度，順時針為正值）。
  - `mode` (int): 插值映射模式（0: 最近鄰反向, 1: 雙線性反向, 2: 正向映射）。
  - `bg_r`, `bg_g`, `bg_b`, `bg_a` (int): 出界區域所要填充的背景顏色 (RGBA)。
* **實作方法**：
  根據角度預先計算 `cos(theta)` 與 `sin(theta)`。遍歷新畫布的每個像素 `(xp, yp)`，計算出原圖實數坐標 `(src_x, src_y)`。
  - **最近鄰插值**：對坐標直接四捨五入，取得對應原圖像素。
  - **雙線性插值**：取得四周相鄰的 4 個像素坐標，依據水平與垂直小數距離進行距離權重乘加計算：
    `w00 = (1-a)*(1-b), w10 = a*(1-b), w01 = (1-a)*b, w11 = a*b`
    進行內插值混色，消除旋轉後物體邊緣的鋸齒。超出邊界的像素則以給定背景色填充。

## 7. 幾何縮放 (scale_image)
* **演算法原理**：
  改變影像寬度與高度。縮放時連續坐標映射為：`src_coord = ((dst_coord + 0.5) * scale) - 0.5`。
  - **最近鄰插值**：將對應的實數坐標向下取整加 0.5（四捨五入），取得最靠近的像素值。
  - **雙線性插值**：藉由目標座標對應原圖的實數坐標，利用其四周 4 個相鄰像素值進行雙線性加權計算。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `newW` (int), `newH` (int): 縮放後的新寬度與新高度。
  - `mode` (int): 縮放插值模式（0: 最近鄰, 1: 雙線性）。
* **實作方法**：
  藉由 `scaleX = w / newW` 與 `scaleY = h / newH` 計算比例。實作中採用「中心補償對齊」，避免幾何中心對齊漂移。對原圖坐標採取 Strict Boundary Clamp：`iy` 限制在 `[0, h - 1]` 區間，`ix` 限制在 `[0, w - 1]` 區間，防範像素訪問越界引起崩潰。

## 8. 手動門檻二值化 (manual_threshold)
* **演算法原理**：
  基於單一閥值 T 的二值分割方法。將多階灰階值轉為純黑白（0 或 255）兩級。
* **參數說明**
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `T` (int): 使用者指定的分隔門檻值（0 到 255）。
* **實作方法**：
  遍歷每個像素。若影像為 `d=1`，當 `pixel_val >= T` 輸出 255，否則輸出 0。若影像為 `d=3` 或 `d=4`，先以 BT.601 公式計算其灰階亮度值，再與 `T` 進行比較，並將 B、G、R 三個通道的值同時設為 0 或 255；若是 `d=4`，則保持原有的 Alpha 透明度值以維護透明背景。

## 9. 大津法二值化 (otsu_threshold)
* **演算法原理**：
  一種自適應二值化方法。其數學準則是透過分析直方圖，自動尋找一個最佳閾值 T_best，使得分割後的背景類別 C1（灰階小於閾值）與前景類別 C2（灰階大於等於閾值）之間的**類間變異數**最大。
  累積背景像素機率權重 `w1(t)`，前景為 `w2(t) = 1 - w1(t)`。
  背景灰階均值 `mu1(t)`，前景灰階均值 `mu2(t)`。
  類間變異數公式：
  `sigma_b^2(t) = w1(t) * w2(t) * (mu1(t) - mu2(t))^2`
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
* **實作方法**：
  1. 統計影像直方圖，並計算所有像素灰階值的加權總和 `total_sum`。
  2. 迴圈疊代 t 從 0 到 254，動態維護背景的累積像素個數 `w_bg` 與背景累積灰階和 `sum_bg`。
  3. 當前閾值 t 下的背景均值為 `mean_bg = sum_bg / w_bg`，前景均值為 `mean_fg = (total_sum - sum_bg) / (TotalPixels - w_bg)`。
  4. 計算當前的類間變異數。若大於歷史最大變異數，則更新最大變異數值，並將 `best_threshold` 設為 t。
  5. 疊代結束後，使用最佳閾值 `best_threshold` 對影像像素執行二值化（0 或 255）映射，輸出至 `g` 中（4 通道保留原有 Alpha）。

## 10. 霍夫直線檢測 (detect_lines_hough)
* **演算法原理**：
  霍夫直線檢測基於直角座標系與極座標參數空間的對偶性。直線表示為：
  `rho = x * cos(theta) + y * sin(theta)`
  其中 `rho` 為原點到直線的垂直距離，`theta` 為其法線與 x 軸的夾角。影像空間中的共線邊緣像素，在參數空間中的正弦曲線會相交於同一點 `(rho, theta)`。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `houghThreshold` (int): 累加器投票門檻值，高於此值才會判定為直線。
* **實作方法**：
  1. 將影像轉為灰階，套用 Sobel 算子計算梯度幅值並以閾值 100 二值化，提取邊緣像素。
  2. 建立大小為 `180 * (2 * D + 1)` 的累加器陣列（`D` 為影像對角線長度），並利用 $0^\circ \sim 179^\circ$ 的 $\sin$ 與 $\cos$ 查表法進行投票。
  3. 對大於等於 `houghThreshold` 的高票點實施 $5 \times 5$ 的非極大值抑制 (NMS)，確保直線不重複偵測。
  4. 將原圖複製至 `g`。對於被判定為直線的參數點，依據角度以 x 或 y 為自變數在 `g` 上疊加繪製紅色直線（`d > 1` 時 BGR 為 `(0, 0, 255)`，單通道 `d == 1` 時繪製白色 `255`）。

## 11. Sobel 邊緣偵測 (detect_sobel)
* **演算法原理**：
  Sobel 算子藉由兩個 $3 \times 3$ 遮罩對影像進行水平與垂直差分，用以逼近影像亮度的一階偏導數：
  - 水平梯度遮罩 $G_x = [-1, 0, 1; -2, 0, 2; -1, 0, 1]$
  - 垂直梯度遮罩 $G_y = [-1, -2, -1; 0, 0, 0; 1, 2, 1]$
  
  對於每一像素點，計算其在 $x$ 與 $y$ 方向上的偏導數 $g_x$ 與 $g_y$ 後，其梯度振幅採用絕對值近似法：
  `mag = |g_x| + |g_y|`
  並限制在 `[0, 255]` 之間。遮罩在差分方向上具有平滑濾波功能，可有效降低對高頻雜訊的敏感度。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數（`d=1` 灰階，`d=3` BGR 彩色）。
  - `g` (int*): 輸出影像陣列指標。
* **實作方法**：
  1. 通道與色彩相容性：若 `d > 1` (彩色)，則先利用 BT.601 公式進行灰階化預處理並存入臨時的 `gray` 陣列；若 `d == 1`，則直接複製像素亮度。
  2. 空間卷積：遍歷內部像素 $y \in [1, h-2], x \in [1, w-2]$，將鄰域像素乘以 $G_x$ 與 $G_y$ 權重求得梯度分量，計算振幅 $mag = |g_x| + |g_y|$，經 `Clamp` 限制在 `[0, 255]` 範圍後寫回輸出陣列 `g`。
  3. 效能優化與邊界安全：完全移除輸入至輸出陣列的全圖資料複製。對影像的最外圈 1 像素邊界統一強制設為 0，落實 Zero Padding 邊界防護，確保 $3 \times 3$ 卷積在鄰域取值時不發生記憶體越界崩潰。

## 12. Canny 邊緣偵測 (detect_canny)
* **演算法原理**：
  Canny 邊緣檢測器遵循低錯誤率、高定位精度與單一響應三項指標。其處理工作流包含：
  1. **高斯降噪**：消除高頻雜訊，使邊緣檢測更穩健。
  2. **計算梯度**：使用 Sobel 算子計算水平 $G_x$ 與垂直 $G_y$ 梯度，以 $M = \sqrt{G_x^2 + G_y^2}$ 逼近一階微分振幅，並以 $\theta = \text{atan2}(G_y, G_x)$ 作為梯度方向。
  3. **非極大值抑制 (NMS)**：沿梯度方向將當前點與兩個相鄰像素進行強度比較，只保留局部強度極大值點，將寬邊緣壓縮至單像素寬度。
  4. **雙閾值與連通追蹤**：將像素區分為強邊緣（$\ge T_{high}$，確認為邊緣點）、弱邊緣（$T_{low} \le \text{mag} < T_{high}$，可能是噪訊或微弱邊緣）與非邊緣（$< T_{low}$，拋棄）。使用顯式堆疊（Stack）執行深度優先搜尋（DFS）追蹤與強邊緣相連的弱邊緣，將其升格為強邊緣點以救援斷裂輪廓，未連通的弱邊緣則清零排除。
* **參數說明**：
  - `f` (int*): 輸入影像一維像素陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出二值化邊緣影像陣列指標。
  - `lowThresh` (double): 低門檻值 (0.0 ~ 255.0)。
  - `highThresh` (double): 高門檻值 (0.0 ~ 255.0)。
* **實作方法**：
  1. 實質灰階化預處理：對彩色 BGR 影像使用 BT.601 權重公式轉為灰階儲存於臨時緩衝區。
  2. 高斯模糊：空間卷積 3x3 高斯核心 `[1 2 1; 2 4 2; 1 2 1] / 16` 以平滑影像。
  3. Sobel 差分：卷積計算 $G_x$ 與 $G_y$ 分量，求得實數幅值與方向角。
  4. NMS 離散方向比對：將角度量化為 $0^\circ, 45^\circ, 90^\circ, 135^\circ$ 四方向，完成局部最大值篩選，非極大值則抑制為 0。
  5. 雙閥值與堆疊 DFS 連通性追蹤：利用分配在 Heap 上的動態陣列 `int* stack` 實作非遞迴式 DFS 連通性搜尋，與遞迴相比，徹底防範大圖可能導致的 Stack Overflow 崩潰風險，保障執行期安全。
  6. 輸出處理與記憶體回收：輸出符合 `d` 通道規格的 0 或 255 像素，並在函式結束時釋放所有暫存陣列以防範 Memory Leak。

## 13. 霍夫圓形檢測 (detect_circles_hough)
* **演算法原理**：
  在二維影像空間中，一個圓形可以表示為：
  (x - a)^2 + (y - b)^2 = r^2
  其中 (a, b) 為圓心，r 為半徑。因此，參數空間是一個三維空間 (a, b, r)。若已知圓周上邊緣點的梯度方向，則圓心 (a, b) 必定位於該邊緣點沿著梯度方向（即法線方向）距離為 r 的位置上。
  藉由將每個邊緣像素的梯度向量投影投票至對應半徑的累積陣列中，可大幅減少投票的盲目性（從原本 360 度投票縮減為沿梯度方向及其鄰近有限角度的投票）。
  當某個 (a, b, r) 的投票數大於預設門檻值，且在 3x3x3 參數空間中是局部最大值時，即可認定檢測到一個圓。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數。
  - `g` (int*): 輸出影像陣列指標。
  - `rMin` (int): 欲偵測的最小半徑。
  - `rMax` (int): 欲偵測的最大半徑。
  - `houghThreshold` (int): 累加器投票門檻值。
  - `lineR`, `lineG`, `lineB` (int): 繪製圓形輪廓的顏色 (RGB)。
* **實作方法**：
  1. 將影像轉為灰階，套用 Sobel 算子計算梯度幅值並以閾值 100 二值化，並保存 dx 與 dy 梯度分量。
  2. 建立三維累加器陣列 `accumulator`，大小為 `(rMax - rMin + 1) * h * w`。
  3. 對於每個邊緣點，計算梯度方向的單位向量 `(ux, uy) = (dx/len, dy/len)`。
  4. 利用 cos(10度) = 0.9848 與 sin(10度) = 0.1736 的常數，在原梯度方向及正負 10 度偏轉的三個方向上，沿著 rMin 到 rMax 的距離計算對應的圓心 (a, b) 並在累加器中加一票。同時也在反方向上投票以檢測亮/暗圓。整個過程完全採用查表與乘加運算，移除了昂貴的 atan2、cos 與 sin 等三角函數呼叫以提升效能。
  5. 在三維參數空間中尋找局部極大值（非極大值抑制），判定為圓的點存入候選列表中。
  6. 將原圖複製至 g，並對每個偵測出的圓形，在輸出畫布 g 上繪製對應半徑的彩色圓形輪廓。

## 14. 二維卷積空間濾波與自訂濾波器 (convolution_filter)
* **演算法原理**：
  空間濾波（鄰域卷積）是透過在影像上滑動一個權重矩陣（核心 Kernel，如 3x3 或 5x5），計算中心像素及其鄰域像素的加權總和。
  本演算法採用補零邊界 (Zero Padding) 處理，超出影像邊界的鄰域像素值一律以 0 代入計算。
  計算公式為：
  `sum = Sum_{i=-kHalf}^{kHalf} Sum_{j=-kHalf}^{kHalf} ( f(x+j, y+i, c) * kernel((i+kHalf)*kSize + (j+kHalf)) )`
  `g(x, y, c) = clamp( sum / divisor + offset )`
  其中 `divisor` 為歸一化除數，`offset` 為亮度偏移量。`clamp` 將輸出值限制在 [0, 255] 區間。
* **參數說明**：
  - `f` (int*): 輸入影像陣列指標。
  - `w` (int), `h` (int): 影像寬度與高度。
  - `d` (int): 影像通道數（d=1 灰階，d=3 BGR 彩色，d=4 BGRA，其中第四通道 Alpha 直接複製而不進行濾波）。
  - `g` (int*): 輸出影像陣列指標。
  - `kernel` (double*): 扁平化的一維雙精度浮點數濾波核心陣列（長度為 kSize * kSize）。
  - `kSize` (int): 核心邊長（3 或 5）。
  - `divisor` (double): 權重除數（不可為 0）。
  - `offset` (double): 亮度偏移值。
* **實作方法**：
  1. 通用卷積核心：外層雙重迴圈遍歷影像像素，內層雙重迴圈遍歷核心鄰域。當鄰域座標出界時，像素亮度值以 0 代入（零邊界防護，對於 3x3 補 1 層，5x5 補 2 層）。
  2. 多通道分流處理：彩色通道（BGR）獨立進行卷積計算與 Clamp 限制；若是 4 通道，Alpha 通道直接從輸入複製到輸出以維護透明度。
  3. C# 前端提供「自訂 3x3 / 5x5 濾波器」介面，預設為恆等核心（中心為 1，其餘為 0），並在執行前實施實質灰階檢查（Color Alert），僅處理灰階影像。
  4. C++ DLL 中的 Sobel 邊緣偵測與 Canny 高斯模糊/梯度計算，皆已重構為直接調用 `convolution_filter` 核心，以提升代碼重用度與邊界安全一致性。

* **系統內置之 5 種預設濾波器詳細說明**：
  1. **平均濾波 3x3 (Mean Filter)**
     - **演算法知識**：一種經典的低通平滑濾波器。透過對 3x3 鄰域內的所有像素值求算術平均，來消除高頻噪訊（如椒鹽噪訊或細微隨機雜訊），但同時會導致影像邊緣與細節變得模糊。
     - **預設核心數值**：
       1  1  1
       1  1  1
       1  1  1
     - **預設參數**：除數 (Divisor) = 9.0，偏移量 (Offset) = 0.0。
     - **運作工作流**：取得 3x3 鄰域的 9 個像素值，加權係數皆為 1，相加後除以 9.0，最後將結果 Clamp 限制在 [0, 255] 後輸出。

  2. **高斯濾波 3x3 (Gaussian Filter)**
     - **演算法知識**：一種基於二維高斯分佈函數的空間低通濾波器。核心像素權重最大，向外呈鐘形衰減。相較於平均濾波，高斯濾波能更自然地平滑隨機噪訊（如高斯噪訊），且對影像輪廓和邊緣的模糊程度較輕。
     - **預設核心數值**：
       1  2  1
       2  4  2
       1  2  1
     - **預設參數**：除數 (Divisor) = 16.0，偏移量 (Offset) = 0.0。
     - **運作工作流**：取得鄰域像素後乘以對應的高斯權重並求和，再除以總權重之和 16.0，最後 Clamp 輸出。

  3. **拉普拉斯銳化 3x3 (Laplacian Filter)**
     - **演算法知識**：一種基於二階微分的空間高通濾波器。利用 8-鄰域拉普拉斯算子來計算像素亮度的二階差分，對於影像中亮度變化的不連續點（如邊緣、線條、噪點）有極強的響應。銳化時通常將提取出的高頻細節疊加回原圖，以增強邊緣對比度。
     - **預設核心數值**：
       -1  -1  -1
       -1   8  -1
       -1  -1  -1
     - **預設參數**：除數 (Divisor) = 1.0，偏移量 (Offset) = 0.0（若需要將負值梯度可視化，可設為 128.0 以呈現中灰色背景的邊緣圖）。
     - **運作工作流**：鄰域周邊 8 個像素權重為 -1，中心像素權重為 8，計算後的加權和除以 1.0 並加上偏移量，最後 Clamp 限制在 [0, 255] 後輸出。

  4. **高斯-拉普拉斯 5x5 (LoG Filter)**
     - **演算法知識**：高斯-拉普拉斯 (Laplacian of Gaussian, LoG) 濾波器。由於拉普拉斯算子對噪訊極為敏感，LoG 演算法先採用高斯平滑濾波去除高頻噪點，再利用拉普拉斯算子進行二階導數零交叉點 (Zero Crossing) 的邊緣偵測，是一種非常穩健的邊緣提取算子。
     - **預設核心數值**：
        0   0  -1   0   0
        0  -1  -2  -1   0
       -1  -2  16  -2  -1
        0  -1  -2  -1   0
        0   0  -1   0   0
     - **預設參數**：除數 (Divisor) = 1.0，偏移量 (Offset) = 128.0（配合 128.0 偏移量可將正負梯度對稱可視化）。
     - **運作工作流**：在 5x5 的範圍內套用上述核心係數進行乘加運算，除以 1.0 後加上 128.0 偏移量，經 Clamp 限制在 [0, 255] 後輸出。

  5. **反銳化遮罩與高提升濾波 (Unsharp Masking / High-Boost Filter)**
     - **演算法知識**：此濾波器是先將原圖進行低通模糊，再用原圖減去該模糊圖得到一個包含細節的高頻「遮罩」(Mask)。最後將原圖乘以一個權重係數 A，並疊加上該細節遮罩。
       數學算式為：g(x,y) = A * f(x,y) - f_blur(x,y)
       當 A = 1.0 時稱為反銳化遮罩；當 A > 1.0 時稱為高提升濾波。
       對於 3x3 鄰域，若採用 3x3 均值濾波作為模糊圖，可推導出對應的合成卷積矩陣：
       中心元素 = 9 * A - 1，其餘 8 個周邊鄰居元素皆為 -1，除數設為 9.0。
       在前端實作中，我們採用更強健的 Subtraction Laplacian 替代算式，其 3x3 合成核心為：
       中心元素 = 8 + (A - 1.0) * 9，其餘 8 個周邊元素皆為 -1，除數為 1.0。
     - **預設參數**：使用者在 UI 滑桿輸入係數 A (1.0 到 3.0，預設為 1.2)。依 A 計算出中心元素，其餘鄰居設為 -1.0。除數 (Divisor) = 1.0，偏移量 (Offset) = 0.0。
     - **運作工作流**：依據使用者選定的比例 A，動態計算 3x3 的核心矩陣數值，隨後調用 `convolution_filter` 進行二維卷積運算，完成影像的局部對比細節增強。
